# Contributing to SaulPOS

Thanks for your interest in improving SaulPOS.

## Before You Start

- Open an issue first for major changes so scope and approach can be aligned.
- Keep pull requests focused and small when possible.
- Follow the existing project structure and coding style.

## Development and Validation

- Use the local development instructions in `README.md`.
- Run relevant tests before opening a pull request.
- Update documentation when behavior, API contracts, or operational flows change.

## Pull Request Checklist

- Clear title and description of the change.
- Linked issue (if applicable).
- Tests added or updated for changed behavior.
- No unrelated refactors mixed into the same PR.
